# Horus Experiment Setup (Cloud)

This folder provides a cloud orchestration setup for **Experiment 1 (hospital scene)** using:
- Isaac Sim launcher from this repo (`fast_isaac_sim.py`)
- ROS 2 Jazzy + CycloneDDS
- Zenoh ROS2DDS bridge with strict topic allowlist
- Compressed camera relay (`image_transport`)

## Files

- `horus_experiment.sh` - operator script (`bootstrap`, `start-exp1`, `stop-exp1`, `status`, `print-local-connect`)
- `config/cyclonedds.xml` - CycloneDDS config used by this setup
- `config/topics_base.txt` - baseline allowlisted topic regex rules
- `config/topics_extra.txt` - editable extra topics for nav/slam and future features
- `config/gen_zenoh_config.sh` - generates `config/zenoh_ros2dds.json5`
- `env/cloud_ros.env` - generated by `bootstrap` (not committed)

## Quick Start (Cloud)

From your cloud home folder:

```bash
git clone https://github.com/RICE-unige/horus_experiment_setup.git
cd ~/horus_experiment_setup
chmod +x horus_experiment.sh
chmod +x config/gen_zenoh_config.sh

./horus_experiment.sh bootstrap
./horus_experiment.sh start-exp1
```

> [!NOTE]
> `bootstrap` now auto-configures the ROS 2 apt repository for Jazzy on Ubuntu when it is missing.
> It also auto-downloads `zenoh-bridge-ros2dds` and auto-clones `isaac-projects` to `~/isaac-projects` when not found.
> Default internal Zenoh listen port is `10000` unless you override `ZENOH_PORT`.

Attach to logs:

```bash
tmux attach -t horus_exp1
```

If your shell is already inside tmux:

```bash
tmux switch-client -t horus_exp1
```

Stop:

```bash
./horus_experiment.sh stop-exp1
```

Status:

```bash
./horus_experiment.sh status
```

## Local Connect

On cloud, print the connect command for your local machine:

```bash
./horus_experiment.sh print-local-connect
```

Expected command shape:

```bash
~/zenoh_internet_bridge/connect_to_cloud.sh <CLOUD_IP> <EXTERNAL_PORT>
```

The helper runs Zenoh in `client` mode with multicast scouting disabled to avoid local UDP bind collisions.

If your cloud provider maps external port to a different internal port:
- keep `ZENOH_PORT` for the cloud bridge listen port (internal),
- set `ZENOH_EXTERNAL_PORT` for the printed local connect command (public mapped port).

## Topic Forwarding (Strict Allowlist)

Zenoh routing config is generated from:
- `config/topics_base.txt`
- `config/topics_extra.txt`

To add future topics (navigation/slam/etc):
1. Edit `config/topics_extra.txt`
2. Restart experiment:

```bash
./horus_experiment.sh stop-exp1
./horus_experiment.sh start-exp1
```

## Verification

On cloud:

```bash
source /opt/ros/jazzy/setup.bash
source ~/horus_experiment_setup/env/cloud_ros.env

ros2 topic list | rg '^/(clock|carter1|carter2|carter3)'
ros2 topic echo /carter1/front_stereo_camera/left/image_raw/compressed --once
ros2 topic echo /carter1/front_2d_lidar/scan --once
```

On local (after connecting Zenoh bridge):

```bash
source /opt/ros/jazzy/setup.bash
ros2 topic list | rg '^/(clock|carter1|carter2|carter3)'
ros2 topic echo /carter1/chassis/odom --once
ros2 topic echo /carter1/front_stereo_camera/left/image_raw/compressed --once
```

## Notes

- This setup does **not** edit scene USD files.
- Default `ROS_DOMAIN_ID` is `5` unless already set before launch.
- Defaults can be overridden with env vars:
  - `HORUS_TMUX_SESSION`
  - `ZENOH_PORT` (internal listen port)
  - `ZENOH_EXTERNAL_PORT` (public/local connect port)
  - `ISAAC_PYTHON`
  - `FAST_ISAAC_SIM`
  - `HOSPITAL_USD`
  - `ZENOH_ROOT`, `ZENOH_BRIDGE`, `ZENOH_CONNECT_SCRIPT`

## Common Cloud Pitfalls

- If your shell user is `root`, `~` resolves to `/root`.
- If your custom project location is not `~/isaac-projects`, set `PROJECT_ROOT` (or `FAST_ISAAC_SIM`/`HOSPITAL_USD`) before running `bootstrap`.
- If public and internal ports differ, set both:
  - `ZENOH_PORT=<internal>`
  - `ZENOH_EXTERNAL_PORT=<public>`
